# Technology Stack

**Analysis Date:** 2026-04-09

## Languages

**Primary:**
- TypeScript 5.8.0 - Server-side MCP implementation and Sonix API client

## Runtime

**Environment:**
- Node.js 22.15 (LTS) - Docker image: `node:22.15-slim`

**Package Manager:**
- npm 10.x (bundled with Node.js 22.15)
- Lockfile: `package-lock.json` present

## Frameworks

**Core:**
- @modelcontextprotocol/sdk 1.29.0 - MCP (Model Context Protocol) server framework for AI tool exposure

**Web/Server:**
- Express 5.2.1 - HTTP server for MCP transport (HTTP mode) and rate limiting

**Runtime Options:**
- MCP SDK provides two transports:
  - **Stdio** (default): Direct stdin/stdout for claude-in-cursor integration
  - **HTTP**: Express-based HTTP server for remote/containerized deployment

**Validation/Schema:**
- Zod 3.25.0 - Runtime schema validation for MCP tool parameters (using zod/v4 API for Standard Schema compatibility)

**Middleware/Security:**
- express-rate-limit 8.3.2 - Rate limiting for HTTP endpoints (60 req/min global, 5 new sessions/min per IP)

## Key Dependencies

**Critical:**
- @modelcontextprotocol/sdk 1.29.0 - Defines MCP server protocol, tool registration, transport layers
- Express 5.2.1 - HTTP server with middleware support for rate limiting, CORS, JSON parsing
- Zod 3.25.0 - Input validation for all tool parameters via Standard Schema integration

**Build/Development:**
- TypeScript 5.8.0 - Language and compiler
- @types/node 22.0.0 - Node.js type definitions
- @types/express 5.0.6 - Express type definitions

## Configuration

**Environment:**
- `SONIX_API_KEY` - Sonix.ai API key (required for both stdio and HTTP modes)
- `TRANSPORT` - Server mode: "stdio" (default) or "http"
- `PORT` - HTTP server port (default: 3000)
- `MAX_SESSIONS` - Maximum concurrent HTTP sessions (default: 100)
- `CORS_ORIGIN` - CORS origin to allow (if not set, CORS is disabled)

**Build:**
- `tsconfig.json` - TypeScript compilation to ES2022, CommonJS modules, strict mode
- Output: `dist/` directory (excluded from git, generated by `npm run build`)

**See also:**
- `.mcp.json` - MCP server configuration for claude-in-cursor integration (stdio mode)
- `Dockerfile` - Two-stage build: compile TypeScript, then run minimal Node.js image

## Platform Requirements

**Development:**
- Node.js 22.15+
- npm (bundled)
- TypeScript installed as dev dependency

**Production:**
- Docker container (recommended): `node:22.15-slim` base image
- Or: Node.js 22.15+ on bare metal
- Environment variable: `SONIX_API_KEY` must be set
- Port 3000 exposed for HTTP mode (if using HTTP transport)
- 30-minute session TTL for HTTP mode clients

**Deployment Target:**
- Docker (primary): Multi-stage build outputs optimized image
- Node.js process manager: PM2, systemd, or similar for stdio mode
- Cloud platforms: Compatible with Railway, Vercel Functions, CloudRun, etc. (via HTTP mode)
